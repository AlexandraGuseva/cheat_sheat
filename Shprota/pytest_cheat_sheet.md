## Pytest — это библиотека, которая находит в вашем проекте специальные функции-тесты, 
## запускает их и проверяет, работает ли ваш код так, как вы задумали.

# 1. Как pytest понимает, что нужно запускать? (Правила имён)
Pytest — парень автоматический, но ленивый. Он ищет тесты только по строгим правилам:
Имя файла должно начинаться с test_ или заканчиваться на _test.py (например: test_rectangle.py).
Имя функции-теста обязательно должно начинаться с test_ (например: def test_area():).
Имя класса с тестами (если используете классы) должно начинаться с Test (например: class TestShapes:).

# 2. Основа любого теста — assert (Проверка)

В pytest не нужно учить сложные команды для проверок. Используется одно стандартное слово 
Python — assert (утверждать). Если условие после него возвращает True — тест пройден. 
Если False — тест упал.

def test_simple_math(): 
# Проверяем равенство 
assert 2 + 2 == 4 
# Проверяем неравенство 
assert 5 > 3 
# Проверяем наличие элемента в списке 
assert "apple" in ["apple", "banana"] 

💡  Как добавить своё сообщение при ошибке:
Если тест упадет, можно вывести понятную подсказку, написав её через запятую:

assert r.area == 15, f"Площадь должна быть 15, а получили {r.area}" 

# 3. Проверка на ошибки (Исключения)
Иногда код должен вызывать ошибку при неправильных данных (например, деление на ноль). 
Мы проверяем это с помощью конструкции with pytest.raises():

import pytest 
def test_division_by_zero(): 
# Проверяем, что код внутри вызовет ошибку ZeroDivisionError 
with pytest.raises(ZeroDivisionError): 1 / 0 

# 4. Фикстуры (@pytest.fixture) — Подготовка данных
Фикстура — это функция, которая готовит чистые данные или объекты перед запуском теста, 
чтобы не создавать их вручную в каждом тесте.

import pytest 
from src.Rectangle import Rectangle 
# 1. Создаем фикстуру, которая возвращает готовый прямоугольник 
@pytest.fixture 
def sample_rectangle(): 
return Rectangle(side_a=3, side_b=5) 
# 2. Передаем имя фикстуры как аргумент в тест 
def test_rectangle_area(sample_rectangle): 
# Pytest сам подставит сюда объект из фикстуры выше 
assert sample_rectangle.area == 15 

# 5. Параметризация (@pytest.mark.parametrize) — Один тест на много данных
Если нужно проверить одну и ту же функцию с десятью разными парами чисел, 
не пишите 10 тестов. Используйте параметризацию:

import pytest 
# Задаем имена переменных и список их значений в формате (вход, ожидаемый_результат) 
@pytest.mark.parametrize("num, expected", [ (2, 4), (3, 9), (4, 16), ]) 
def test_square(num, expected): 
assert num ** 2 == expected 

# 6. Главные команды для Терминала
Откройте вкладку Terminal внизу PyCharm и запускайте тесты одной из команд:
pytest — Запустить абсолютно все тесты во всём проекте.
pytest tests/test_Rectangle.py — Запустить тесты только из конкретного файла.
pytest -k test_area — Запустить только те тесты, в имени которых есть фраза test_area.
pytest -v — Подробный (verbose) вывод: покажет имена всех запущенных функций, а не просто точки.
pytest -x — Остановить тестирование сразу же после первой упавшей ошибки (чтобы не ждать остальные).

# 7. Файл настроек pytest.ini
Этот файл создается в корне проекта, чтобы не вводить длинные команды в терминал вручную каждый раз.

[pytest] 
# addopts — автоматические флаги для терминала. 
# -v (подробно), -v (показывать принты внутри тестов) 
addopts = -v -s 
# Здесь регистрируются свои маркеры, чтобы pytest не ругался на них warnings 
markers = 
    smoke: быстрые критически важные тесты 
    slow: долгие интеграционные тесты
