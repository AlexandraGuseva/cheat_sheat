# 🐙  2. Работа с Git (в терминале и PyCharm)

### Горячие клавиши Git в PyCharm
* `Cmd` + `K` — Открыть окно **Commit** (фиксация изменений).
* `Cmd` + `Shift` + `K` — Открыть окно **Push** (отправка в удаленный репозиторий).
* `Cmd` + `T` — **Update Project** (подтянуть изменения из origin / git pull).
* `Ctrl` + `V` — Меню VCS (быстрые операции с Git).

### Консольные команды Git
## Шаг 1: Начало работы и настройка. Эти команды выполняются один раз при установке Git или создании нового проекта.
git config --global user.name "Ваше Имя" — Указывает ваше имя для коммитов.
git config --global user.email "your@email.com" — Привязывает вашу почту к коммитам.
git init — Создает новый пустой репозиторий Git в текущей папке.
git clone [ссылка_на_репозиторий] — Скачивает существующий проект с GitHub на компьютер.

## Шаг 2: Работа с изменениями (Каждый день). Эти команды используются постоянно в процессе написания кода.
git status — Показывает текущее состояние файлов (какие изменены, какие еще не отслеживаются).
git add [имя_файла] — Добавляет конкретный файл в список для сохранения (индекс).
git add . — Добавляет все измененные и новые файлы в текущей папке.
git commit -m "Название коммита" — Сохраняет добавленные изменения на компьютере и подписывает их.

## Шаг 3: Синхронизация с GitHub (Remote). Команды для обмена кодом с удаленным сервером.
git remote add origin [ссылка] — Связывает ваш локальный проект с пустым репозиторием на GitHub.
git push -u origin main — Отправляет ваш код на GitHub (флаг -u связывает ветки, в следующий раз можно писать просто git push).
git pull — Скачивает новые изменения из GitHub и сразу объединяет их с вашим локальным кодом.
git fetch — Просто скачивает информацию о новых коммитах из GitHub, но не внедряет их в ваш код (безопасная проверка).

## Шаг 4: Работа с ветками (Branches). Нужно для создания безопасных копий проекта, чтобы тестировать новые фичи.
git branch — Показывает список всех локальных веток.
git branch [имя_ветки] — Создает новую ветку с указанным именем.
git checkout [имя_ветки] — Переключает вас на указанную ветку.
git checkout -b [имя_ветки] — Создает новую ветку и сразу переключает на нее (совмещает две предыдущие команды).
git merge [имя_ветки] — Находясь в главной ветке (например, main), вливает в неё изменения из указанной ветки.

## Шаг 5: Отмена и просмотр истории. Если нужно посмотреть, что было сделано, или исправить ошибку.
git log — Показывает историю всех коммитов (для выхода из этого режима нажмите клавишу Q).
git log --oneline — Показывает историю коммитов в компактном виде (одна строка на коммит).
git reset --soft HEAD~1 — Отменяет самый последний коммит, но оставляет все ваши изменения в файлах нетронутыми (удобно, если ошиблись в тексте сообщения).

### Чтобы связать уже созданный локальный проект из PyCharm с новым репозиторием на GitHub, выполните следующие шаги:
## Шаг 1. Создайте пустой репозиторий на GitHub
   1. Зайдите на сайт github.com и войдите в свой аккаунт.
   2. В правом верхнем углу нажмите кнопку + ➔ New repository.
   3. Введите Repository name (имя проекта).
   4. Важно: Оставьте галочки Add a README file, Add .gitignore и Choose a license пустыми. 
      Репозиторий должен быть абсолютно чистым.
   5. Нажмите кнопку Create repository.
   6. Скопируйте ссылку на созданный репозиторий из адресной строки браузера 
      (или нажмите кнопку копирования рядом с полем HTTPS). Ссылка выглядит так: https://github.com.

## Шаг 2. Инициализируйте Git в PyCharm (если это не сделано)
Если ваш проект еще не отслеживается системой Git:
   1. В верхнем меню PyCharm выберите VCS ➔ Enable Version Control Integration... 
      (или Git ➔ Initialize Local Git Repository...).
   2. В выпадающем списке выберите Git и нажмите OK.
   3. Все файлы проекта станут красными — это нормально, Git их заметил.

## Шаг 3. Привяжите адрес GitHub к проекту
   1. В верхнем меню перейдите в Git ➔ Manage Remotes....
   2. В открывшемся маленьком окне нажмите на значок + (плюс).
   3. В поле URL вставьте ссылку, которую вы скопировали на Шаге 1. 
      Поле Name оставьте стандартным — origin.
   4. Нажмите OK.

## Шаг 4. Сделайте первый коммит и отправку
   1. Нажмите Ctrl + K (на Windows) или Cmd + K (на macOS), чтобы открыть вкладку Commit.
   2. Отметьте галочками файлы, которые хотите отправить (обычно все, кроме папки .idea и venv).
   3. Внизу в поле сообщения напишите, например: Initial commit.
   4. Нажмите на стрелочку рядом с кнопкой Commit и выберите Commit and Push... 
      (либо сначала нажмите Commit, а затем Ctrl+Shift+K / Cmd+Shift+K для вызова окна Push).
   5. В открывшемся окне нажмите Push.

Если PyCharm попросит авторизоваться, выберите Log in via GitHub... — откроется браузер, 
где нужно будет в один клик разрешить PyCharm доступ к вашему аккаунту.

### Чтобы папка .idea (и другие временные файлы) больше никогда не появлялись в этом списке и не мешали вам при коммитах, нужно использовать специальный файл-фильтр, который называется .gitignore.
Вот как его правильно настроить за одну минуту:

## Шаг 1. Создайте файл .gitignore
В левой панели PyCharm, где отображаются папки вашего проекта, кликните правой кнопкой мыши 
по самому верхнему корню проекта (PythonProject).
Выберите New ➔ File.
Введите точное название файла: .gitignore (обязательно с точкой в самом начале и без расширений 
вроде .txt) и нажмите Enter.
Если PyCharm спросит "Do you want to add the following file to Git?", нажмите Cancel (Отмена) 
или Don't Ask Again.

## Шаг 2. Добавьте в него правила исключения
Файл откроется как обычный текстовый документ. Вставьте в него следующие строчки (это стандартный 
набор для Python-проектов):
.idea/ 
*.iml 
*.xml 
venv/ 
.venv/ 
ENV/  
__pycache__/ 
*.pyc 

## Шаг 3. Сохраните изменения
После того как вы вставили этот текст, Git мгновенно применит правила. Вы можете сразу открыть 
вкладку Commit слева — вы увидите, что папка .idea, файлы .xml и папка виртуального окружения 
venv автоматически исчезли из списка доступных для отправки файлов. Там останется только 
сам файл .gitignore и ваши файлы с кодом/шпаргалками.
